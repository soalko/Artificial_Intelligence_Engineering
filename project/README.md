---

## 1. Паспорт проекта

- **Название проекта:** `Прогнозирование спроса на товары в розничной сети`
- **Автор:** `Солдатов Александр Константинович`
- **Группа:** `ЭФБО-04-24`
- **Контакт:** `@soldalex`

- **Краткое описание (2-4 предложения):**  
  Проект решает задачу прогнозирования ежедневных продаж (спроса) товаров в розничной сети.  
  Используется открытый датасет Demand forecasting dataset (Kaggle) с 16 признаками, включая цену, инвентарь,
  промоакции, погоду и сезонность.  
  Реализован полный пайплайн: EDA, инжиниринг признаков (лаги, скользящие средние, признаки даты), обучение
  XGBoost-модели и REST API для получения прогнозов.

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
    - `exp01_eda_baseline.ipynb` – EDA и простые baseline‑модели.
- `src/` – основной код проекта:
    - `data/` – загрузка и предобработка данных (`load_data.py`, `preprocess.py`);
    - `features/` – построение признаков (`build_features.py`);
    - `models/` – обучение модели (`train.py`);
    - `service/` – FastAPI сервис (`app.py`);
    - `utils/` – вспомогательные функции (`helpers.py`).
- `data/` – демонстрационные/учебные данные:
    - `raw/` – исходный CSV (не хранится в репозитории, скачивается по инструкции);
    - `processed/` – предобработанные данные и данные с признаками.
- `configs/` – конфигурационные файлы:
    - `config.yaml` – пути, гиперпараметры, настройки признаков;
    - `.env.example` – шаблон для переменных окружения.
- `tests/` – тесты:
    - `test_data_loading.py` – проверка наличия данных и модели;
    - `test_model.py` – проверка работоспособности модели.
- `artifacts/` – сохранённые модели, метрики, отчёты.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10`
- Установленные системные зависимости: `libomp` для macOS (для XGBoost)
- Доступ в интернет для скачивания датасета (или ручная загрузка)

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (опционально, но рекомендуется)
python -m venv .venv

# Активировать окружение:
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
# Если на macOS возникает ошибка с XGBoost, как у меня)), выполните:
brew install libomp
```

## 4. Как запустить проект

Опишите, как именно воспроизвести основные сценарии использования проекта.

### 4.1. Подготовка данных

```
cd project
source .venv/bin/activate
python -m src.data.load_data       # скачивает датасет (или даёт инструкцию)
python -m src.data.preprocess      # очистка и сохранение processed.csv
python -m src.features.build_features  # создание признаков (лаги, скользящие и т.д.)
```

### 4.2. Обучение модели

```
python -m src.models.train
```

Модель и метрики сохранятся в artifacts/.

### 4.3. Запуск сервиса (API)

```
python -m src.service.app
```

Сервис поднимается на порту 8000.
Доступные эндпоинты:

- POST /predict – получить прогноз продаж.

Пример тела запроса:

```
{"Store_ID": "S001, "Product_ID": "P0004", "Date": "2019-01-15"}
```

Ответ:

```
{"predicted_units_sold": 80.78}
```

- GET /health – проверка работоспособности.

Проверка через curl:

```
curl -X POST "http://localhost:8000/predict" \
-H "Content-Type: application/json" \
-d '{"Store_ID":"S001","Product_ID":"P0004","Date":"2019-01-15"}'

```

### 4.4. Запуск с помощью Docker

```
cd project
docker build -t demand-forecast .
docker run -p 8000:8000 demand-forecast
Или с использованием docker-compose:
docker-compose up --build
```

Сервис будет доступен по тому же адресу http://localhost:8000.

## 5. Данные

- Источник: открытый датасет Demand forecasting dataset с Kaggle.
- Данные содержат 76 000 строк, 16 колонок: Date, Store ID, Product ID, Category, Region, Inventory Level, Units Sold (
  целевая переменная), Units Ordered, Price, Discount, Weather Condition, Promotion, Competitor Pricing, Seasonality,
  Epidemic, Demand.
- В репозитории не хранятся большие файлы. Для загрузки данных используется скрипт src/data/load_data.py (автоматически
  через kagglehub или с инструкцией для ручного скачивания).
- После предобработки и построения признаков данные сохраняются в data/processed/.

## 8. Ограничения и дальнейшая работа

### Ограничения:

- Сервис не сохраняет историю запросов и не имеет авторизации.
- Признаки строятся на основе всей исторической выборки, что в реальном времени может быть заменено на кеширование.
- Модель не обновляется онлайн (требуется переобучение).

### Дальнейшая работа:

- Добавить доверительные интервалы к прогнозам.
- Реализовать пакетный режим инференса (CSV → CSV).
- Внедрить мониторинг (Prometheus + Grafana) и логирование запросов.
