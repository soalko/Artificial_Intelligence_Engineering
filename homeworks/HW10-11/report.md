# HW10-11 – компьютерное зрение в PyTorch: CNN, transfer learning, detection/segmentation

## 1. Кратко: что сделано

- Для части A выбран датасет STL10, так как он стандартный, предсказуемый и позволяет показать эффект аргументаций.
- Для части B выбран Pascal VOC detection потому что есть готовая предобученная модель, остается визуализировать.
- В части A сравнивались 4 эксперимента: CNN без аргументаций, CNN с аргументациями, ResNet18 с замороженным backbone,
  ResNet18 с частичным fine-tuning – разморожены layer4 и fc. В части B сравнивались два режима интерфейса: с порогом
  score_threshold = 0.3 и score_threshold = 0.7

## 3. Данные

### 3.1. Часть A: классификация

- Датасет: `STL10`
- Разделение: 5000 train / 8000 test / свой val из train 80/20
- Базовые transforms: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]
- Augmentation transforms: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]
- Комментарий (2-4 предложения): STL10 содержит 10 классов объектов (самолёт, птица, автомобиль, кот и др.) с
  разрешением 96×96 пикселей. Изображения обладают высоким разнообразием и содержат сложные текстуры. Это позволяет
  наглядно продемонстрировать влияние аугментаций и transfer learning на качество классификации.

### 3.2. Часть B: structured vision

- Датасет: `Pascal VOC`
- Трек: `detection`
- Что считается ground truth: bounding boxes и (координаты xmin, ymin, xmax, ymax) и метка класса для каждого
  обнаруженного объекта.
- Какие предсказания использовались: Предсказания получены от предобученной модели fasterrcnn_resnet50_fpn из
  torchvision. Для каждого изображения модель возвращает набор bounding boxes с соответствующими оценками уверенности (
  scores) и метками классов.
- Комментарий (2-4 предложения): Pascal VOC – эталонный датасет для детекции объектов, содержащий хорошо
  сбалансированные классы, разметку в едином формате и умеренный размер, что позволяет проводить эксперименты с
  предобученными моделями без больших вычислительных затрат. Это позволяет анализировать влияние гиперпараметра на
  метрики качества.

## 5. Часть B: постановка задачи и режимы оценки (V1-V2)

### Если выбран detection track

- Модель:
- V1: `score_threshold = 0.3`
- V2: `score_threshold = 0.7`
- Как считался IoU: Для каждой пары предсказанный бокс — ground truth вычисляется Intersection over Union (IoU) по
  стандартной формуле: IoU= площадь объединения / площадь пересечения, где пересечение и объединение рассчитываются по
  координатам углов боксов. Сопоставление предсказаний с ground truth выполняется по принципу «каждому GT — лучший по
  IoU предсказанный бокс» при условии IoU >= 0.5
- Как считались precision / recall: TP – количество предсказанных боксов, успешно сопоставленных с каким-либо GT (с
  IoU ≥ 0.5), FP – количество предсказанных боксов, не сопоставленных ни с одним GT, FN – количество GT-боксов, не
  сопоставленных ни с одним предсказанием. Далее: Precision = TP/(TP + FP)
